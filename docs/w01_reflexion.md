# Reflexión W01

## ¿Qué haría que tu sistema sea “no confiable” aunque el código corra?
Un sistema puede parecer funcional y aun así ser poco confiable si no valida la calidad del dato de entrada, si no deja trazabilidad de qué archivo usó, o si produce resultados reproducibles solo por accidente. Que el código corra no garantiza que las conclusiones sean correctas.

## ¿Qué documento te gustaría encontrar si heredas el proyecto de otro equipo?
Me gustaría encontrar un documento corto pero claro con el contrato de datos, las decisiones de diseño, las reglas de calidad mínimas y evidencia de una corrida reciente. Ese tipo de documentación reduce mucho el tiempo de entendimiento inicial.
